# 利用Cloudflare R2 + Workers搭建在线网盘


[汉化修改自/longern/FlareDrive](https://github.com/longern/FlareDrive)

增加了权限系统，支持多管理员，分别授权目录

cloudflare R2是一个文件储存系统，配合Cloudflare Workers可以实现这样一个网盘系统

### 搭建教程


1. fork该仓库
2. 前往Cloudflare R2新建一个R2储存桶，并前往储存桶设置，允许公开访问，复制**公共存储桶 URL**
3. 前往Cloudflare Pages新建一个站点，选择连接到Git

4.选择刚刚fork的仓库，点击开始设置
5.项目名称可以修改，其他项目保持默认不动

6.展开环境变量，添加

| 变量名称| 值|
| --- | --- |
| PUBURL | 复制的**公共存储桶URL** |
| GUEST | public/ |
| admin:123456 | * |
| user1:123456 | user1/,userPublic/ |

以此类推，GUEST代表游客的允许写入目录

管理员则以`账号:密码`的形式设置，值代表其允许写入的目录，用`,`隔开，**请勿在前后加逗号，否则会授予所有目录的写入权限**

设置好后点击**开始部署**

7.前往Pages->cloudflare-r2-oss->设置->函数->R2 存储桶绑定,绑定R2存储桶,变量名称`BUCKET`

8.在部署页面重新部署即可

## 🔒 安全权限配置（重要更新）

### 读写权限分离

本版本已修复安全漏洞，现在支持读取和写入权限的分离控制：

#### 读取权限配置
在原有环境变量基础上，需要为每个用户添加读取权限：

| 变量名称 | 值 | 说明 |
|---------|----| ---- |
| admin:123456_READ | * | 管理员完全读取权限 |
| user1:123456_READ | user1/,public/ | 用户1读取权限 |

#### 完整权限配置示例
```bash
# 管理员（完全权限）
admin:123456=*
admin:123456_READ=*

# 普通用户（受限权限）
user1:123456=temp/,uploads/
user1:123456_READ=uploads/,shared/

# 游客权限（可选）
GUEST=public/temp/        # 游客写入权限
GUEST_READ=public/        # 游客读取权限
```

### 安全改进说明

1. **解决信息泄露**: 未授权用户无法查看目录结构和文件列表
2. **流量控制**: 文件下载需要认证，防止恶意爬取
3. **权限分离**: 可独立控制读取和写入权限

### 环境变量格式

- **写入权限**: `username:password=目录1,目录2`
- **读取权限**: `username:password_READ=目录1,目录2`
- `*` 表示所有目录权限
- 如不设置读取权限，用户将无法查看文件列表

### 故障排除

如果遇到认证问题：
1. 检查是否同时设置了读取和写入权限
2. 确认环境变量名格式正确（无多余空格）
3. 等待1-2分钟让环境变量生效

## 🗂️ 递归删除文件夹功能（新增）

### 功能特性

本版本新增了强大的文件夹递归删除功能：

- 📁 **完整删除**: 删除文件夹时会自动删除所有文件和子文件夹
- 🔄 **无限深度**: 支持任意深度的嵌套文件夹结构
- ⚡ **批量处理**: 并行删除提高大文件夹删除速度
- 🎯 **智能提示**: 区分文件和文件夹的删除确认对话框
- 💬 **操作反馈**: 显示加载状态和成功/失败提示消息

### 使用方法

#### 删除文件夹
1. 右键点击要删除的文件夹
2. 选择"删除"选项
3. 确认删除对话框会显示：
   ```
   确定要删除文件夹 "my-folder" 吗？
   ⚠️ 这将递归删除文件夹中的所有文件和子文件夹，此操作不可恢复！
   ```
4. 点击"确定"开始删除
5. 等待删除完成（大文件夹需要时间）
6. 成功后会显示绿色提示消息

#### 性能基准
| 文件数量 | 预计删除时间 | 备注 |
|---------|-------------|------|
| < 100 | 1-3 秒 | 快速删除 |
| 100-1000 | 3-10 秒 | 正常速度 |
| 1000-10000 | 10-30 秒 | 需要等待 |
| > 10000 | 30+ 秒 | 建议分批处理 |

### ⚠️ 重要注意事项

1. **删除不可恢复**: R2 存储没有回收站，删除后无法恢复
2. **备份重要数据**: 删除前请备份重要文件
3. **权限要求**: 需要对整个文件夹路径的写入权限
4. **大文件处理**: 包含大量文件的文件夹删除需要耐心等待

### 故障排除

- **删除超时**: 检查网络连接，大文件可能需要更长时间
- **部分残留**: 可能是权限问题，再次执行删除操作
- **确认框不显示**: 检查浏览器是否阻止弹窗

### 技术改进

- **后端**: 实现了 `deleteFolderRecursively` 函数，支持分页处理和错误容错
- **前端**: 添加了智能确认对话框、加载状态和 Toast 消息提示
- **性能**: 使用并行删除和分页处理优化大文件夹删除速度
